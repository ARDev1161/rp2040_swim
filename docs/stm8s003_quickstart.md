# STM8S003F3 / STM8S103F3 Quickstart

1. Build and flash the RP2040-Zero firmware.
2. Wire GPIO2 to SWIM through a 220-1000 ohm resistor.
3. Wire GPIO3 to NRST.
4. Connect common GND.
5. Use a 3.3 V target or level shifter. RP2040 GPIO is not 5 V tolerant.

First real target test:

```sh
python3 host/rp2040_swim.py enter-debug --port /dev/ttyACM0
python3 host/rp2040_swim.py read --port /dev/ttyACM0 --addr 0x8000 --len 16 --out dump.bin
```

Entry waveform capture without an STM8:

Terminal 1:

```sh
cd /mnt/raid/projects/Keyboard/rp2040_swim_test
python3 host/validation_runner.py passive-capture \
  --analyzer-port /dev/ttyACM1 \
  --rate 10m \
  --duration 10ms
```

Terminal 2:

```sh
cd /mnt/raid/projects/Keyboard/rp2040_swim
python3 host/rp2040_swim.py entry-waveform \
  --port /dev/ttyACM0 \
  --pullup \
  --delay-ms 1000
```

Expected: NRST is low during the waveform, SWIM starts high, initial low is about
16 us, then four 1 kHz pulses, four 2 kHz pulses, and SWIM ends high/released.
The entry waveform is generated by PIO by default; C bit-bang is fallback/debug
only.

Probe after `enter-debug` succeeds:

```sh
python3 host/rp2040_swim.py probe --port /dev/ttyACM0
```

Read the first 256 bytes of program flash:

```sh
python3 host/rp2040_swim.py read --addr 0x8000 --len 256 --out dump.bin
```

Flash command shape:

```sh
python3 host/rp2040_swim.py flash --device stm8s003f3 --file firmware.ihx --verify --reset
```

The current firmware intentionally refuses erase/program commands until the flash
sequence is hardware-validated.

Unlock flash or EEPROM without erasing:

```sh
python3 host/rp2040_swim.py unlock --device stm8s003f3 --flash
python3 host/rp2040_swim.py unlock --device stm8s003f3 --eeprom
```
